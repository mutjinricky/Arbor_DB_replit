# 드라이어드 (Dryad) 프로젝트 인수인계서

## 1. 프로젝트 개요 (Project Overview)
**드라이어드 (Dryad)**는 Arbor Aware라고도 불리며, 체계적인 수목 재고 관리 및 관련 프로젝트(가지치기, 병해충 방제 등)를 효율적으로 수행하기 위한 웹 애플리케이션입니다.
관리자는 대시보드를 통해 수목 상태를 모니터링하고, 지도 기반 인터페이스로 수목 위치를 파악하며, 유지보수 프로젝트의 예산과 진행 상황을 관리할 수 있습니다.

## 3. 주요 데이터 모델 (Key Data Models)

핵심 비즈니스 로직에서 사용되는 주요 인터페이스 정의입니다.

### 3.1 수목 데이터 (`TreeData`)
`src/pages/TreeInventory.tsx` 및 GeoJSON 파싱 로직에서 사용됩니다.
```typescript
interface TreeData {
  id: string;          // 수목 고유 ID (예: "TREE-001")
  species: string;     // 수종
  lat: number;         // 위도
  lng: number;         // 경도
  risk: string;        // 위험도 ("high" | "medium" | "low")
  height: number;      // 수고 (m)
  district: string;    // 행정구역
  inspection: string;  // 최근 점검일
}
```

### 3.2 예산 상세 (`FinalBudget`)
`src/lib/utils.ts`의 예산 산출 로직에서 반환되는 구조체입니다.
```typescript
interface FinalBudget {
  // 직접비
  directMaterials: number; // 재료비
  directLabor: number;     // 노무비
  directExpenses: number;  // 경비
  
  // 간접비 및 보험료
  indirectLabor: number;   // 간접노무비
  healthInsurance: number; // 건강보험료
  // ... 기타 4대 보험 및 안전관리비
  
  // 최종 합계
  generalAdmin: number;    // 일반관리비 (6%)
  profit: number;          // 이윤 (15%)
  vat: number;             // 부가세 (10%)
  finalTotal: number;      // 최종 견적 금액
}
```

## 4. 폴더 구조 및 주요 로직 상세 (Directory & Logic Deep Dive)

### 4.1 디렉토리 구조
- **`src/components`**: `TreeMap`, `TreeLayer` 등 지도 관련 컴포넌트와 공통 UI가 위치합니다.
- **`src/pages`**: 각 라우트별 메인 페이지 컴포넌트입니다.
- **`src/lib`**: `utils.ts` (비즈니스 로직 및 유틸리티)가 핵심입니다.
- **`public/data`**: `trees.geojson`, `trees.json` 등 정적 데이터 파일이 위치합니다.

### 4.2 핵심 로직 상세

#### **A. 지도 시각화 레이어 (`src/components/TreeLayer.jsx`)**
Mapbox Expression을 사용하여 데이터 기반 스타일링을 구현했습니다.
- **Zoom-dependent Styling**: 줌 레벨에 따라 마커의 크기(`circle-radius`)가 동적으로 변합니다. (줌 14: 5px -> 줌 25: 50px)
- **Risk Color Mapping**: `properties.risk` 값에 따라 색상을 자동 지정합니다.
```javascript
const riskColorMapping = [
  'match', ['get', 'risk'],
  'high', riskColors.riskHigh,    // 빨강
  'medium', riskColors.riskMedium,// 주황
  'low', riskColors.riskLow,      // 초록
  riskColors.riskOther
];
```

#### **B. 예산 산출 엔진 (`src/lib/utils.ts`)**
수목 유지보수 견적을 위한 복잡한 계산 로직이 구현되어 있습니다.
- **직접비 계산**: 가지치기(직경별 단가), 외과수술(면적/부피별), 영양공급(약제/직경별) 비용을 합산합니다.
- **제비율 적용**: 직접비를 바탕으로 간접노무비, 경비, 보험료, 일반관리비, 이윤 등을 법정 요율에 맞춰 역산출(`calculateFinalBudget`)합니다.

#### **C. 라우팅 및 전역 설정 (`App.tsx`)**
- `BrowserRouter` 내에 레이아웃(`TopNav`)이 정의되어 있어 모든 페이지에 네비게이션이 유지됩니다.
- `Toaster` 컴포넌트가 최상위에 배치되어 전역 알림을 처리합니다.

#### **D. 유틸리티 및 훅**
- **`cn` (`src/lib/utils.ts`)**: `clsx`와 `tailwind-merge`를 결합하여 CSS 클래스 충돌 없이 조건부 스타일링을 도와줍니다.
- **`useToast` (`src/hooks/use-toast.ts`)**: 알림 팝업을 관리하는 훅으로, `toast()` 함수를 통해 어디서든 알림을 띄울 수 있습니다.

## 5. 환경 변수 및 설정 (Configuration)

루트 디렉토리의 `.env` 파일 설정이 필수적입니다.

```env
# Mapbox 지도 렌더링을 위한 필수 토큰
VITE_MAPBOX_TOKEN=your_mapbox_access_token_here
```
> **주의**: 이 토큰이 설정되지 않으면 지도 영역이 회색으로 표시되며 "Mapbox 토큰 필요" 경고가 뜹니다.


## 6. 공개 데이터 (Public Data) 관리

프로젝트 루트의 `public/data` 폴더에는 애플리케이션 동작에 필수적인 정적 데이터 파일들이 포함되어 있습니다. 백엔드 API가 없는 현재 단계에서 이 파일들은 DB 역할을 대신합니다.

### 6.1 주요 데이터 파일
| 파일 경로 | 형식 | 용도 | 사용되는 컴포넌트/페이지 |
| :--- | :--- | :--- | :--- |
| `/data/trees.geojson` | GeoJSON | **지도 시각화**: Mapbox 레이어(`TreeLayer`)에 수목 위치와 상태를 점으로 표시하는 데 사용됩니다. | `TreeInventory.tsx`, `RequestForm.tsx`, `TreeSelector.tsx` |
| `/data/trees.json` | JSON | **상세 정보 조회**: 수목 ID를 키로 하는 객체 형태(`Record<string, TreeData>`)로, 특정 수목의 상세 스펙을 조회할 때 사용됩니다. | `TreeInventory.tsx` (목록뷰), `CreateProject.tsx`, `RequestForm.tsx` |
| `/data/tree_images/` | Images | **수목 이미지**: 수목 상세 프로필이나 모달창에 표시되는 예시 이미지 모음입니다. | `TreeProfileModal.tsx`, `CreateProject.tsx` |

### 6.2 데이터 사용 방식
- **FETCH**: 애플리케이션 로드 시(`useEffect`) `fetch('/data/trees.json')` 형태로 데이터를 비동기로 불러와 `useState`에 저장합니다.
- **수정**: 수목 데이터를 추가하거나 수정하려면 이 파일들을 직접 편집해야 합니다. (개발/테스트 단계 한정)
- **배포**: 빌드 시 `dist` 폴더로 함께 복사되므로 별도의 설정 없이 프로덕션 환경에서도 접근 가능합니다.
