# 스마트 수목 관리 — 프로젝트 설명서

> 코드 검증일: 2026-04-26
> 화면 표시명: **스마트 수목 관리** (이천시) · 저장소·인수인계서상의 별칭: **드라이어드(Dryad)**

---

## 한 줄로 말하면

**이천시의 가로수·공원수 2,985그루를 한 화면에서 관리하는 웹 대시보드입니다.**

종이 대장이나 엑셀로 흩어져 있던 수목 정보를 지도 위에 모아 보고, 위험한 나무는 색깔로 즉시 알아볼 수 있게 만든 도구라고 생각하시면 됩니다.

---

## 누가, 왜 쓰나요?

- **이천시 공원녹지과 담당 공무원** — 어느 나무가 위험한지, 언제 방제(약 치는 일)를 해야 하는지 한눈에 확인
- **현장 작업자** — 작업 지시서로 어떤 나무에 어떤 작업이 필요한지 확인
- **민원 담당자** — "○○동 가로수 가지가 처졌어요" 같은 민원을 해당 나무 정보에 바로 연결

종이로 관리하면 "이 나무 작년에 점검했나?"를 알 수 없지만, 이 시스템은 모든 이력을 한 나무 단위로 추적합니다.

## 핵심 기능

### 1️⃣ 수목 지도 — 4가지 모드

지도에 2,985그루가 점으로 찍히고, 보고 싶은 정보에 따라 색상이 바뀝니다.

- **기본** — 위치만 표시 (색상 단일)
- **수목 위험도** — 쓰러질 위험이 큰 나무는 빨강 (4단계: 경/중/심/극심)
- **해충 방제** — 곧 약을 쳐야 하는 나무는 빨강 (5단계: 안정/사전준비/예찰/방제준비/실행)
- **토양** — 흙 상태가 나쁜 나무는 빨강 (5단계: A~E). 토양 모드에서는 지도에 구역(zone) 폴리곤도 함께 표시됩니다.

부가 기능:
- **수목 ID 검색** → 입력 후 "지도 이동" 버튼을 누르면 해당 나무 위치로 자동 줌(19배)
- **필터 패널** — 수종·위험 등급·해충 등급·토양 등급을 AND 조합 (예: "산수유 + 위험도 심")
- **호버 툴팁** — 마우스 올린 나무의 등급·점수·해충명 미리보기
- **목록 보기** — 같은 데이터를 표 형태로 전환

### 2️⃣ 위험도 자동 계산

사람이 일일이 판단하지 않고, 코드가 공식으로 계산합니다.

- **수목 위험도** ([src/lib/riskCalculations.ts](src/lib/riskCalculations.ts)): 육안진단 / 부후도 / 기울기 — 3개 지표 중 **최고 위험 등급**을 최종 등급으로 채택
- **K-UTSI 토양 점수**: 6가지 항목(토양 경도·토성·유기물·pH·EC·CEC) 각 5점 만점 → 30점 합산 후 100점 환산
  - A(≥90) / B(75~89) / C(60~74) / D(40~59) / E(<40)
- **해충 방제 시기**: 기상청 일평균기온의 **유효적산온도(DD)** 누적치로 산출
  - 대상 해충: 복숭아순나방(7.2°C, 260DD) / 꽃매미(8.14°C, 355DD) / 갈색날개매미충(12.1°C, 202DD)

### 3️⃣ 방제 달력

- 1년 12달 어느 시점에 어떤 해충이 활동하는지 그래프로 표시
- **복숭아순나방**은 1~4세대까지 다세대로 추적 (DD 214/660/1380/1950)
- **돌발해충 이력 데이터**: 미국선녀벌레·매미나방 등 침입 해충의 연도별 발생 심각도와 과거 방제 사업 이력
- 기상청 API 연동 시 실시간 DD, 미연동 시 평년값으로 자동 대체

### 4️⃣ 작업 지시서 관리

대시보드/수목 지도에서 "작업 지시서 생성"을 누르면 모달이 뜹니다.

- 대상 수목 선택
- 작업 유형 체크: **가지치기 / 병해충 방제 / 영양제 처방 / 외과수술 / 수목 제거**
- 예상 비용 자동 합산 (총합 표시)
- 프로젝트명 + 메모 입력 후 "RFQ 전송"

대시보드 위젯은 전체 작업 지시서를 **입찰 중 / 진행 중 / 검토 대기 / 완료** 4단계로 집계해 막대 그래프로 보여줍니다.

> ⚠️ "RFQ 전송" 버튼은 현재 데모 단계입니다. 실제 외부 시스템 전송은 일어나지 않고 토스트 알림만 뜹니다. 추후 백엔드 연동 후 실제 전송이 가능해집니다.

### 5️⃣ 수목 상세 모달 — 7개 탭

지도나 목록에서 나무 한 그루를 클릭하면 다음 탭이 열립니다.

| 탭 | 내용 |
|---|---|
| **개요** | 수고·흉고직경·수령·최근 점검일 |
| **위험성** | 3대 지표(육안진단/부후도/기울기) 점수와 등급 + K-UTSI 토양 6항목 |
| **이력** | 점검·관리 이력 타임라인 (관리/관찰 필터) |
| **민원** | 접수된 시민 민원 목록 (날짜·유형·처리 상태) |
| **비용** | 누적 비용 + 유형별 비중 + 연평균 |
| **사진** | 점검·유지보수 작업 사진 갤러리 |
| **편집** | 현장에서 수고·흉고직경·상처 면적·공동 깊이 + 자유 태그 직접 수정 |

편집 내용은 **해당 브라우저에만 저장**됩니다(localStorage). K-UTSI 점수 재계산에는 반영되지 않습니다.

### 6️⃣ 사업이력 (사업 단위 관리)

- 연도·지역·유형별 사업 목록을 표로 관리
- 사업 하나에 들어있는 정보: 사업명·기간·예산·집행액·시공업체·담당부서·연결 수목·완료 사진·설계 도면 첨부
- 도면 파일 업로드 시 자동으로 수목 정보를 추출하는 기능 ([src/lib/designExtraction.ts](src/lib/designExtraction.ts) 호출)
- 대시보드 **최근 사업** 패널에는 진행 중인 사업의 D-day·진행률 막대·상태(착공/작업중/작업완료/준공)가 표시됩니다.

### 7️⃣ 해충 방제 D-day 알림

대시보드 상단에 자동으로 떠오르는 배너:

- 🚨 **긴급** — 7일 이내 방제 필요
- ⚠️ **주의** — 30일 이내
- ℹ️ **안내** — 60일 이내

각 배너는 X 버튼으로 닫을 수 있습니다 (세션 동안만).

## 기술 구조 (간단히)

- **순수 프런트엔드 웹 앱입니다.** 별도 서버 없이 브라우저에서만 돕니다.
- **수목 데이터**: [public/data/trees.json](public/data/trees.json) (2,985그루, 객체 형태) + [public/data/trees.geojson](public/data/trees.geojson) (지도용)
- **구역 데이터**: [public/data/zones.geojson](public/data/zones.geojson) (토양 분석용)
- **편집·태그·작업 상태**는 브라우저 저장소(localStorage)에 남아 있어, 같은 PC·같은 브라우저에서는 새로고침해도 유지됩니다 (다른 PC와는 공유되지 않음).
- **기상 데이터**는 기상청 ASOS API에서 일평균기온을 받아옵니다. API 키가 없으면 [src/lib/weatherApi.ts](src/lib/weatherApi.ts)의 평년값 시뮬레이션으로 자동 대체됩니다.
- 지도 우측 상단의 **"기상청 연동" / "평년값 시뮬레이션"** 배지로 현재 어느 데이터를 쓰는지 확인할 수 있습니다.

```
브라우저 ─▶ React 웹앱
            ├── 지도 (Mapbox GL)
            ├── 수목 데이터 (trees.json + trees.geojson)
            ├── 구역 데이터 (zones.geojson)
            └── 기상 데이터 (기상청 API ↔ 평년값 자동 전환)
```

주요 기술 스택: React 18 · TypeScript · Vite 5 · Tailwind CSS · shadcn/ui · TanStack Query · React Router · Mapbox GL · Recharts.

## 실행 방법

```bash
npm install      # 처음 한 번
npm run dev      # 개발 서버 시작 (http://localhost:5000)
npm run build    # 배포용 빌드
```

`.env` 파일에 두 가지 키:

| 키 | 필수 | 설명 |
|---|---|---|
| `VITE_MAPBOX_TOKEN` | ✅ 필수 | 없으면 지도 영역에 회색 안내 메시지만 표시 |
| `VITE_KMA_API_KEY` | ⬜ 권장 | 없으면 평년값 시뮬레이션으로 동작 (배지가 노랑색으로 표시) |

기상청 API 키는 [공공데이터포털](https://data.go.kr)에서 "지상(종관, ASOS) 일자료 조회서비스"를 검색해 발급받습니다. 개발 환경에서는 [vite.config.ts](vite.config.ts)에 정의된 `/api/kma` 프록시를 통해 호출되며, 운영 배포 시에는 별도 서버사이드 프록시가 필요합니다.

## 알려진 한계 / 앞으로 할 일

| 항목 | 현재 상태 |
|---|---|
| 기상청 API 키 | 미발급 → 평년값 시뮬레이션으로 대체 중 |
| 토양 실측 데이터 | 구역(zone)별 대표값 → 그루별 정밀 토양 조사 필요 |
| 민원 데이터 | 모의 데이터 → 이천시 민원 시스템 연동 필요 |
| 작업 지시서 RFQ 전송 | 토스트 알림만 표시 → 실제 전송은 백엔드 구축 후 |
| 데이터 영구 저장 | 브라우저 localStorage → 백엔드 DB 구축 시 다기기 동기화 가능 |
| 수목 데이터 갱신 | trees.json 수동 편집 → CSV 업로드 또는 백엔드 연동 필요 |
| 대시보드 KPI | 일부 하드코딩 값 (고위험 153주·예산 40% 등) → 실데이터 연결 필요 |
