## 1. 프로젝트 개요

**드라이어드 (Dryad)** 는 이천시 가로수 및 공원수 **2,985그루**를 체계적으로 관리하는 스마트 수목 관리 웹 대시보드입니다.  
지도 기반 시각화, 위험도 자동 산출, 해충 방제 타이밍 계산, 토양 건전성 평가, 민원 로그, 작업 알림 기능을 제공합니다.

### 1.1 기술 스택

| 구분 | 기술 | 용도 |
|---|---|---|
| **Framework** | React 18 + Vite 5 | SPA 개발 |
| **Language** | TypeScript | 타입 안정성 |
| **Styling** | Tailwind CSS + shadcn/ui | UI 컴포넌트 |
| **Map** | Mapbox GL + react-map-gl | 인터랙티브 지도 |
| **Data Fetch** | TanStack Query v5 | 비동기 데이터 관리 |
| **Routing** | React Router DOM v6 | 클라이언트 라우팅 |
| **Charts** | Recharts | 데이터 시각화 |
| **Build** | Vite (port 5000) | 개발 서버 + 프록시 |

## 4. 데이터 모델

### 4.1 TreeFullData (trees.json)

```typescript
interface TreeFullData {
  id: string;           // 수목 ID (예: "1349")
  diameter: number;     // 흉고직경 (cm)
  height: number;       // 수고 (m)
  lat: number;          // 위도
  lng: number;          // 경도
  district: string;     // 소재지 (예: "경사리 도로")
  damage_area: number;  // 피해 면적 (m²)
  cavity_depth: number; // 수간 공동 깊이 (cm)
  ice_damage: boolean;  // 설해 피해 여부
  need_nutrient: boolean; // 영양 공급 필요 여부
  risk: string;         // 원본 위험 등급 ("high" | "medium" | "low")
  age: number;          // 수령 (년)
  inspection: string;   // 최근 점검일 (YYYY-MM-DD)
  species: string;      // 수종 (예: "산수유 나무")
}
```

## 5. 구현 기능 상세

### C-01: 인터랙티브 수목 지도 – 고급 필터

**위치**: `src/pages/TreeInventory.tsx`

- 지도 모드 3가지: 수목 위험도(IQTRI) / 해충 방제 / 토양(K-UTSI)
- 필터 패널: 수종, IQTRI 등급, 해충 등급, 토양 등급 (AND 조합)
- 수목 ID 검색 + 지도 자동 이동 (줌 19배)
- Hover 팝업: 해당 모드 기준 점수·등급·해충명 표시
- 필터된 수목만 지도에 표시, 목록과 동기화

### C-02: 수목 상세 프로필 – 민원 로그

**위치**: `src/components/TreeProfileModal.tsx`

- 수목 프로필 모달에 "민원" 탭 추가
- 탭 헤더에 민원 건수 배지 표시
- 날짜·내용·처리상태(처리중/완료)·신고자 목록

### C-03: 작업 시작/종료 알림

**위치**: `src/components/WorkNotifications.tsx`

- 역할 전환: 작업자(worker) / 공무원(official)
- 작업자: 시작 버튼 → 메모 입력 → 종료 버튼 (시각 자동 기록)
- 공무원: 알림 수신 목록, 확인 버튼
- 당일 17:00 자동 종료 (setTimeout)
- 상태 localStorage 영속화

### C-04: 위험도 자동 산출

**위치**: `src/lib/riskCalculations.ts`

#### C-04A: IQTRI
- `D × T × I` 공식 (결함 × 표적 × 영향)
- trees.json `risk`, `damage_area`, `ice_damage`, `cavity_depth`, `diameter`, `district` 활용

#### C-04B: 해충 유효적산온도 (DD)
- 발육영점온도: 복숭아순나방 7.2°C / 꽃매미 8.14°C / 갈색날개매미충 12.1°C
- **기상청 ASOS API 실시간 연동** (`VITE_KMA_API_KEY` 설정 시)
- API 미설정 시 이천 평년값 시뮬레이션으로 fallback
- 수목별 미기후 편차 ±5 DD 반영
- Vite 프록시: `/api/kma/` → `https://apis.data.go.kr/`

#### C-04C: K-UTSI 토양 건전성 지수
- 15개 지표, 4개 범주, 2단계 가중치
- **trees.json 실제 필드를 K-UTSI 지표에 매핑** (단순 난수 아님)
  - `diameter` → ERA(유효뿌리공간)
  - `damage_area`, `cavity_depth` → H(경도), INF(기반시설간섭)
  - `need_nutrient` → SOM, pH, STR
  - `district` → PER, SUR, TRA, EC
  - `ice_damage` → PPT, WAS
  - `age` → HOR(A층깊이)
- 공식: K-UTSI = (총합 / 80) × 100
- A(≥80) / B(65~79) / C(50~64) / D(35~49) / E(<35)

| 프로덕션 기상 API | Vite 프록시는 개발 전용 | 배포 시 서버 프록시 별도 구축 필요 |
